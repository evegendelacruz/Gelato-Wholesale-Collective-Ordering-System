🚀 FIRST TIME SETUP (Do this once)

1. OPEN THE TEMPLATE
   • Locate: Sample.btw (in public/assets/)
   • Double-click to open in BarTender

2. CONFIGURE PRINTER
   • File → Page Setup
   • Printer: Toshiba B-415
   • Paper Size: 90mm x 50mm (landscape)
   • Click OK

3. SET UP DATABASE CONNECTION
   • File → Database Connection Setup
   • Click "Add Database Connection"
   • Select "Text File" or "CSV File"
   • Browse and select: SAMPLE_TEST_DATA.csv (in this folder)
   • Settings:
     ✓ First row contains field names
     ✓ Delimiter: Comma
     ✓ Text qualifier: Double quote
   • Click Next → Finish

4. MAP THE FIELDS TO TEXT OBJECTS

   For each text box on your label, right-click → Properties → Data Source:

   ┌─────────────────────────────────────────────────────┐
   │ CSV Column      │ Maps to Label Element            │
   ├─────────────────────────────────────────────────────┤
   │ CompanyName     → Client's business name            │
   │ ProductName     → Product description               │
   │ Allergens       → Allergen warnings                 │
   │ BestBefore      → Expiry date (DD/MM/YYYY)          │
   │ BatchNumber     → Production batch code             │
   │ Ingredients     → Ingredient list (optional)        │
   └─────────────────────────────────────────────────────┘

   ⚠️ IMPORTANT: Field names are CASE-SENSITIVE!

5. TEST PRINT
   • File → Print Preview
   • Check that all data appears correctly
   • If OK: File → Print
   • Set Copies = 1 (quantity is already in the CSV)
   • Click Print

⚠️ COMMON MISTAKES TO AVOID

❌ Setting Copies > 1 in print dialog
   → CSV already contains correct quantity
   → Always use Copies = 1

❌ Field names don't match exactly
   → Use exact names: CompanyName, ProductName, etc.
   → Check capitalization

❌ CSV file is open in Excel while trying to print
   → Close Excel before printing from BarTender

❌ Wrong printer selected
   → Always select: Toshiba B-415

❌ Wrong paper size
   → Must be: 90mm x 50mm landscape

🔧 TROUBLESHOOTING

PROBLEM: "Cannot connect to database"
SOLUTION:
• Make sure CSV file exists and isn't open in another program
• Check file path is correct
• Try moving CSV to simpler path (e.g., C:\Labels\)

PROBLEM: "Data fields are empty"
SOLUTION:
• Right-click text object → Properties → Data Source
• Verify field name matches CSV column exactly
• Refresh database connection

PROBLEM: "Wrong number of labels printing"
SOLUTION:
• The CSV already has one row per label
• Set Print Copies = 1
• Each CSV row = one label

PROBLEM: "Labels are wrong size"
SOLUTION:
• File → Page Setup → Stock Size: 90mm x 50mm
• Orientation: Landscape
• Save the template
